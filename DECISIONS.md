# Decisions

Log of choices not fully specified by INSTRUCTIONS.md / SPEC.md, per the "note the decision" rule.

- **2026-07-23 ‚Äî Product name/folder:** `alonapp` after the chosen domain alonapp.com; product name "Alon".
- **2026-07-23 ‚Äî Next.js 16.2:** `create-next-app@latest` produced Next 16 (spec says 14+). Kept ‚Äî newest App Router, TS strict on by default.
- **2026-07-23 ‚Äî RLS tests in Vitest, not pgTAP:** TypeScript suite (`tests/rls.test.ts`) using `@supabase/supabase-js`; service-role client seeds fixtures, per-role user clients assert isolation. Same `npm test` entry point locally and in CI, no extra pg tooling.
- **2026-07-23 ‚Äî Supabase CLI as npm devDependency:** no global install needed; `npx supabase ‚Ä¶` works identically locally and in CI.
- **2026-07-23 ‚Äî Client JWT claims via custom access token auth hook:** Postgres function `public.custom_access_token(event jsonb)` reads `public.user_roles` and injects `user_role` + `client_id` claims; registered in `supabase/config.toml`. Freelancers get `user_role: 'freelancer'` on workspace creation; client users get `user_role: 'client'` + their `client_id`.
- **2026-07-23 ‚Äî Client file uploads deferred:** INSTRUCTIONS.md's RLS section says client sessions may INSERT on `messages` only (and the merge-required tests assert exactly that), so storage/table policies give clients read-only file access. `files.uploaded_by = 'client'` stays in the schema for when uploads land in v2.
- **2026-07-23 ‚Äî JWT claim named `user_role`, not `role`:** the reserved `role` claim maps to the Postgres role (`authenticated`) and can't safely carry `'client'`; policies check `auth.jwt() ->> 'user_role'` instead.
- **2026-07-23 ‚Äî Production deployed early (before milestone 3):** hosted Supabase project `fvsupccynnhqeywpuukg` in `ap-southeast-1` (Singapore, closest region to PH); Vercel project `alonapp` with GitHub push-to-deploy from `judegamboa/alonapp` main; live at https://www.alonapp.com (apex alonapp.com redirects to www; alonapp.vercel.app also serves). All 5 migrations pushed; auth config (custom access token hook) synced via `supabase config push`. DB password lives with the owner only (resettable in the dashboard), service-role key only in Vercel production env.
- **2026-07-23 ‚Äî Email debounce is per-thread, direction-agnostic:** `notifyNewMessage` skips if the thread's `last_emailed_at` is within 15 min, regardless of who sent the last message. Recipient is chosen by author: a freelancer message notifies the client's email; a client message notifies the workspace owner's email. `lib/email.ts` no-ops (logs) when `RESEND_API_KEY` is unset so local flows never break.
- **2026-07-23 ‚Äî freelancer-auth email template exists but Supabase sends the live ones:** the 6th React Email template is built, but Supabase Auth still delivers freelancer confirmation/reset mail (Mailpit locally). Wiring our template requires a Supabase "send email" auth hook ‚Äî deferred; the other 5 templates fire from our own server actions.
- **2026-07-23 ‚Äî Local dev via Supabase CLI (Docker):** no hosted project yet; hosted project + Vercel wiring deferred to deployment.
- **2026-07-23 ‚Äî A 7th email template (`payment-request`):** SPEC.md locks 6 templates and none covers payment requests, but a request the client never learns about is useless ‚Äî the portal is pull-only. Added `emails/payment-request.tsx` + `notifyPaymentRequest`, fired once on creation (no debounce; requests are deliberate, low-frequency acts, unlike messages). The CTA deep-links into the portal rather than the pasted payment URL, matching `files-shared`'s "link into the portal, never the payload" rule.
- **2026-07-23 ‚Äî "Mark as paid" is freelancer-only:** SPEC.md says "manual mark as paid toggle" without naming who toggles. The milestone-2 RLS grants clients `SELECT` on `payment_requests` and nothing else, and INSTRUCTIONS.md scopes client writes to messages ‚Äî so the freelancer marks payment received. `tests/rls.test.ts` now pins this (client update/delete both affect 0 rows).
- **2026-07-23 ‚Äî Currency as short dropdown + free code:** USD/PHP/AUD cover the corridor in SPEC.md (PH freelancers, US/AU clients); a 3-letter `currency_other` field overrides the dropdown for everyone else. `formatAmount` (`lib/currency.ts`) wraps `Intl.NumberFormat` and falls back to `CODE 0.00` for codes Intl rejects, so an odd currency degrades to plain text instead of throwing.
- **2026-07-23 ‚Äî Milestone 8 needed no migration:** `payment_requests` (table, index, RLS policies, grants) shipped back in milestone 2 alongside the rest of the schema; milestone 8 was app-layer only.
